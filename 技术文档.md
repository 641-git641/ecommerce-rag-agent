## 1. 项目概述

### 1.1 项目定位

本系统是一个基于 **检索增强生成（Retrieval-Augmented Generation, RAG）** 技术的智能电商导购系统。系统整合向量检索、关键词检索、大语言模型（LLM）生成、知识图谱扩展及多轮对话记忆等能力，支持文本/图片/语音三种输入模态，为用户提供精准的商品咨询、筛选推荐、对比决策等导购服务。

### 1.2 核心功能矩阵

| 功能模块 | 描述 | 实现方式 |
|---------|------|---------|
| 单轮模糊推荐 | "推荐一双跑鞋" | RAG 直通管线 |
| 条件筛选 | "推荐500元以内的防晒霜" | 图谱硬过滤 + 槽位累积 |
| 多轮细化追问 | R1: "跑鞋" → R2: "轻量的" → R3: "预算500" | 记忆系统增量累积 |
| 对比决策 | "MacBook Air 和 Pro 怎么选" | CompareTool 并行检索 |
| 场景化组合推荐 | "三亚度假带什么" | ComboTool 多品类并行 |
| 主动反问澄清 | 信息不足时的追问 | ClarifyTool + force_clarify |
| 反选排除 | "不要华为"、"换一款别的" | graph_filter $nin + 品牌排除 |
| 拍照找货 | 上传图片找同款 | Vision + RAG 混合检索 |
| 语音问答 | 语音输入输出 | ASR(客户端) + TTS(服务端) |
| 购物车 | 多轮结果加入购物车 | Go 网关 + MySQL 持久化 |

### 1.3 运行环境

| 组件 | 技术栈 |
|------|--------|
| 部署方式 | Docker Compose 一键编排 / 本地直接运行 |
| 运行时 | Python 3.11 / Go 1.25 |
| Web 框架 | FastAPI (Python) / Gin (Go 网关) |
| 数据库 | ChromaDB 0.4.x (向量) + MariaDB 11 (业务) |
| LLM 网关 | DashScope (通义千问) 兼容 OpenAI 协议 |
| 客户端 | Android (Kotlin + Jetpack) |

## 2. 技术选型

### 2.1 技术栈全景

| 分层 | 选型 | 说明 |
|------|------|------|
| 应用框架 | Python 3.11 + FastAPI + Uvicorn | 异步高性能 RESTful API |
| API 网关 | Go 1.25 + Gin + httputil.ReverseProxy | SSE 流代理 + 会话/购物车管理 |
| 容器化 | Docker + Docker Compose | 一键部署，开箱即用 |
| 向量数据库 | ChromaDB 0.4.x（持久化模式） | 轻量级，零运维 |
| 嵌入模型 | DashScope `text-embedding-v4` | 1024 维，8192 上下文 |
| 视觉嵌入 | DashScope `tongyi-embedding-vision-flash` | 768 维，以图搜图 |
| 大语言模型 | Qwen 系列 (`qwen-turbo`) | 兼容 OpenAI 协议，超时 180s |
| 重排序 | DashScope `qwen3-rerank` | Cross-Encoder API |
| 关键词检索 | 自研 BM25（纯 Python） | 零依赖，懒加载 |
| 知识图谱 | 自研轻量 JSON 内存图谱 | 零依赖图数据库 |
| 语音识别 | 客户端 Android SpeechRecognizer | 不消耗服务端 API |
| 语音合成 | DashScope `cosyvoice-v3-plus` | 仅语音输入场景开启 |
| 视觉服务 | DashScope `qwen-vl-plus` | 图片商品描述 |
| 记忆系统 | 自研二层记忆 | 实体桥接 + 槽位累积 |
| Agent 框架 | 自研 ReAct 模式 | 意图驱动路由 + 工具编排 |
| 业务数据库 | MariaDB 11 (Docker) / MySQL 8.0 | 购物车 + 会话持久化 |

### 2.2 模型名称一览

本系统所有 AI 模型均通过 DashScope（阿里云百炼平台）调用，一个 API Key 即可使用全部能力。

| 配置环境变量 | 模型名称 | 用途 | 维度/参数 |
|-------------|---------|------|----------|
| `LLM_MODEL_NAME` | `qwen-turbo` | 文本生成（对话、推荐、意图分类、商品对比） | max_tokens=512 |
| `EMBEDDING_MODEL_NAME` | `text-embedding-v4` | 文本向量化（商品文档 → 1024 维向量） | 1024 维 / 8192 ctx |
| `RERANKER_MODEL_NAME` | `qwen3-rerank` | Cross-Encoder 精排（检索结果重排序） | — |
| `VISION_MODEL_NAME` | `qwen-vl-plus` | 图片内容理解（拍照找货、图片描述） | — |
| `ASR_MODEL_NAME` | `fun-asr-realtime` | 服务端语音识别（multimodal 接口备用） | — |
| `TTS_MODEL_NAME` | `cosyvoice-v3-plus` | 语音合成（AI 回复 → 语音播报） | voice=`longanyang` |
| `VISION_EMBEDDING_MODEL_NAME` | `tongyi-embedding-vision-flash` | 图片向量化（以图搜图） | 768 维 |

> **注意**：语音识别（ASR）优先使用 Android 客户端内置 `SpeechRecognizer`，不消耗服务端 API。`fun-asr-realtime` 仅作为 WebSocket 多模态接口的备用方案。

## 4. 核心链路设计

### 4.1 RAG 主流程

```
用户提问
  │
  ├─(1)─ 意图识别 + 元数据过滤 ─────────────────────────  < 1ms
  │      关键词匹配 5 类: price_sku/faq/review/basic/marketing
  │
  ├─(2)─ 多查询扩展（短问题 ≤15字跳过） ────────────────  0-2s (LLM)
  │      合并 HyDE + 多角度扩展为单次 LLM 调用
  │
  ├─(3)─ 多路并行检索 ──────────────────────────────────  ~1s
  │      向量检索(k=3) + BM25 关键词(k=3) × N queries
  │      ThreadPoolExecutor(max_workers=4)
  │
  ├─(4)─ 加权 RRF 融合 ─────────────────────────────────  < 5ms
  │      score = Σ(weight × 1/(rank+60))
  │      vector:0.6 / bm25:0.4，失败自动降级
  │
  ├─(5)─ 后置硬过滤 ────────────────────────────────────  < 1ms
  │      内存层 $in/$nin 过滤（ChromaDB 不可靠兜底）
  │
  ├─(6)─ 多层重排序 ────────────────────────────────────  < 500ms
  │      元数据加权 → Cross-Encoder qwen3-rerank → MMR(λ=0.7)
  │
  ├─(7)─ 知识上下文构建 ─────────────────────────────────  < 5ms
  │      FAQ特殊展开 + 价格/ID补全 + 滑动窗口 2800 字截断
  │
  ├─(8)─ 图谱一跳展开 ───────────────────────────────────  < 1ms
  │      同子品类竞品信息注入（字典遍历）
  │
  ├─(9)─ LLM 生成 ──────────────────────────────────────  2-5s
  │      Prompt 渲染 + chat_stream(max_tokens=512)
  │
  └─(10)─ 后处理 ──────────────────────────────────────── < 50ms
         结构化 JSON 解析 → recommendations + voice_friendly
```

### 4.2 查询扩展策略

| 策略 | 触发条件 | 作用 | 是否调 LLM |
|------|---------|------|:--:|
| 合并扩展 (HyDE + 多角度) | 问题 > 15 字 | 生成假设文档 + 多角度查询 | 1 次 |
| 短问题直通 | 问题 ≤ 15 字 | 原词直搜，跳过 LLM | 否 |
| `skip_query_expansion=True` | Agent 工具内调用 | 跳过扩展，直接检索 | 否 |
| `skip_generation=True` | Compare/Combo 工具 | 跳过重排+生成，回原文 | 否 |

### 4.3 检索融合与重排序体系

```
                       多路检索
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    向量检索结果     BM25检索结果     向量检索结果
    (query_1)        (query_1)       (query_2...)
          │               │               │
          └───────────────┼───────────────┘
                          │
                  加权 RRF 融合 ──────── 公式: Σ w×1/(rank+60)
                (vector 0.6 / BM25 0.4)
                          │
                   后置硬过滤 ─────────── $in/$nin 内存层兜底
                          │
                   元数据权威加权 ─────── FAQ+20% / review-10%
                          │
               Cross-Encoder API 精排 ── qwen3-rerank
                          │
                  MMR 多样性重排 ──────── λ=0.7
                          │
                    Top-K 结果 (k=3)
```

### 4.4 后退提问机制（Step-Back）

当 LLM 生成质量不足时，自动触发两轮后退检索：

| 轮次 | 策略 | 描述 |
|:--:|------|------|
| 1 | 关键词重搜 | 抽取核心关键词，移除元数据过滤，重新检索 |
| 2 | 无过滤检索 | 使用原始问题，彻底移除 filter，最大召回 |
| 终止 | 放弃 | 两轮后仍未改善则返回原始结果 |

### 5.2 建表 DDL

```sql
-- 数据库
CREATE DATABASE IF NOT EXISTS ecommerce_cart
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 会话表
CREATE TABLE IF NOT EXISTS sessions (
    id         VARCHAR(36)  PRIMARY KEY,
    title      VARCHAR(200) NOT NULL DEFAULT '新对话',
    created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    msg_count  INT          NOT NULL DEFAULT 0
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 会话消息表
CREATE TABLE IF NOT EXISTS session_messages (
    id         BIGINT       AUTO_INCREMENT PRIMARY KEY,
    session_id VARCHAR(36)  NOT NULL,
    role       VARCHAR(16)  NOT NULL,
    content    TEXT         NOT NULL,
    cards      TEXT,
    voice_url  VARCHAR(500),
    created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX      idx_session_id (session_id),
    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 购物车表
CREATE TABLE IF NOT EXISTS cart_items (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    user_id     VARCHAR(64)  NOT NULL COMMENT '用户ID（= session_id）',
    product_id  VARCHAR(64)  NOT NULL COMMENT '商品编码',
    product_name VARCHAR(255) NOT NULL,
    price       DECIMAL(10,2) DEFAULT 0.00,
    quantity    INT           DEFAULT 1,
    selected    TINYINT(1)    DEFAULT 1 COMMENT '1=选中 0=取消',
    created_at  TIMESTAMP     DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_product (user_id, product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单主表
CREATE TABLE IF NOT EXISTS orders (
    id             INT AUTO_INCREMENT PRIMARY KEY,
    user_id        VARCHAR(64)  NOT NULL,
    order_no       VARCHAR(32)  NOT NULL UNIQUE,
    address        VARCHAR(500) DEFAULT '',
    contact_name   VARCHAR(100) DEFAULT '',
    contact_phone  VARCHAR(20)  DEFAULT '',
    total_amount   DECIMAL(10,2) DEFAULT 0.00,
    status         VARCHAR(20)  DEFAULT 'pending' COMMENT 'pending/confirmed',
    created_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单明细表
CREATE TABLE IF NOT EXISTS order_items (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    order_id     INT          NOT NULL,
    product_id   VARCHAR(64)  NOT NULL,
    product_name VARCHAR(255) NOT NULL,
    price        DECIMAL(10,2) DEFAULT 0.00,
    quantity     INT          DEFAULT 1,
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 8. Agent 智能体设计

### 8.1 意图驱动路由

```
用户输入 → classify_query()
  │
  ├─ intent="simple"/"exclude" → RAG 直通快速路径
  │   不经过 ReAct，直接 graph_filter → RAG 检索 → 生成
  │
  ├─ intent="compare" → CompareTool 并行检索 → LLM 对比
  │   提取商品名 → 双路 search → 组装卡片 → 总结
  │
  ├─ intent="combo" → ComboTool 场景多品类并行
  │   场景映射(度假/旅行/户外...) → 5路并行 RAG → 组装
  │
  ├─ intent="cart" → CartTool
  │   解析"第N个"/"加入购物车" → 从记忆获取 pid → Go API
  │
  └─ 其他 → ReAct 循环 (最多5轮)
      planner.LLM决策 → tool执行 → observe → 下一轮
```

### 8.2 ReAct 决策循环

```
for step in 1..MAX_REACT_STEPS(5):
    decision = planner.decide(llm, query, history, intent)
    
    if decision.action == "finish":
        return decision.answer
    
    tool = tools[decision.action]
    result = tool.execute(decision.tool_args, session_id)
    history.append(step)
    
    if 信息充足: break

return llm.chat(上下文汇总)
```

### 8.3 工具集

| 工具 | 功能 | skip_generation | 并发 |
|------|------|:---:|:---:|
| `RecommendTool` | 智能推荐 | 否 | - |
| `CompareTool` | 双商品对比 | **是** | 并行 2 路 |
| `ComboTool` | 场景组合推荐 | **是** | 并行 N 路 |
| `ClarifyTool` | 信息不足反问 | - | - |
| `CartTool` | 购物车操作 | - | HTTP |
| `SearchTool` | 知识检索 | **是** | - |

> `skip_generation=True` 表示跳过重排序 + LLM 生成，直接返回原始检索上下文，大幅提速。

### 9.1 二层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 实体记忆桥接（Entity Bridge）                    │
│  记住最近 3 轮推荐的商品ID → 图谱查同款/搭配 → 注入上下文    │
│  触发词：同款/类似的/替代/搭配/还有吗/再推荐                 │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 会话槽位（SessionSlots）                        │
│  增量累积结构化需求                                       │
│    category  — 品类（跑鞋/手机/防晒霜...）                 │
│    attrs[]   — 属性（轻量/透气/防水...），FIFO 最多 5 个    │
│    budget    — 预算上限（取最小值）                         │
│    brand     — 品牌偏好 / 排除品牌                        │
│    exclude[] — 排除项（品牌/日系/含酒精...）                │
│  自动构建增强：原问题 + 槽位序列化                          │
└─────────────────────────────────────────────────────────┘
```

### 9.2 话题切换检测

| 用户输入 | has_category | has_attr/budget | 判定 | 行为 |
|---------|:---:|:---:|------|------|
| "要轻量的" | ❌ | ✅ (轻量) | 细化追问 | **保留品类** |
| "预算500以内" | ❌ | ✅ (500以内) | 细化追问 | **保留品类** |
| "还有别的吗" | ❌ | ❌ | 模糊追问 | **保留品类 + 排除品牌** |
| "推荐平板电脑" | ✅ (平板≠跑鞋) | - | 话题切换 | **清空旧 state** |

## 12. RAG 管线详解

### 12.1 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `RETRIEVAL_K` | 3 | 每路检索返回文档数 |
| `rrf_k` | 60 | RRF 平滑参数 |
| `hybrid_vector_weight` | 0.6 | 向量检索权重 |
| `hybrid_bm25_weight` | 0.4 | BM25 权重 |
| `mmr_lambda` | 0.7 | MMR 相关度-多样性平衡 |
| `KNOWLEDGE_LIMIT` | 2800 | 上下文字符上限 |
| `LLM_MAX_RETRIES` | 2 | LLM 调用重试次数 |
| `LLM_TIMEOUT` | 180s | OpenAI 客户端超时 |
| `generate max_tokens` | 512 | chat_stream 最大输出 |

### 12.2 容错设计

| 层级 | 机制 | 描述 |
|------|------|------|
| 检索融合 | RRF 降级 | RRF 失败 → 普通合并去重 |
| 重排序 | 逐层降级 | 每层重排失败自动跳过 |
| LLM 调用 | 退避重试 | 失败重试 2 次，退避 2s→4s |
| JSON 解析 | 多层容错 | markdown fence 剥离 + 尾部逗号清理 + JSONDecodeError 兜底 |
| 图谱过滤 | 空结果快速通道 | 过滤后无候选 → 跳过 RAG，直接返回提示 |
| ChromaDB | 后置硬过滤 | $in/$nin 不可靠 → 内存层二次过滤 |

### 12.3 TTS 节能策略

```
文本查询 → enable_tts=False → 不调 cosyvoice
图片查询 → enable_tts=False → 不调 cosyvoice
语音输入 → enable_tts=True  → 唯一触发 TTS
```

SSE `voice` 事件始终下发 `text` 字段，客户端可用 Android TTS 本地播报，不消耗服务端 API。

### 13.5 性能基线

| 指标 | 典型值 | 说明 |
|------|--------|------|
| 意图分类 | < 1ms | 纯关键词匹配 |
| 查询扩展 | 0-2s | 合并调用 1 次 LLM（短问跳过） |
| 向量检索(k=3) | ~300ms | ChromaDB 单路 |
| BM25 检索(k=3) | ~50ms | 纯内存 TF-IDF |
| RRF 融合 | < 5ms | 纯计算 |
| Cross-Encoder 精排 | ~300ms | DashScope API |
| LLM 生成流 | 2-5s | qwen-turbo, max_tokens=512 |
| 图谱过滤 | < 5ms | 内存字典遍历 ~100 商品 |
| **端到端** | **3-8s** | 含 LLM 网络延迟 |
